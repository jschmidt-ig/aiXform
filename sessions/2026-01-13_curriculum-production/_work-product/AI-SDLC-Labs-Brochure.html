Choose Your Track

Select based on whether your team is building new services or improving existing ones.

Greenfield Track

For teams building new modules or services from scratch.

  • Weeks 2-4: Build new endpoints/services in sandbox
  • Focus on tests and docs alongside new code
  • Weeks 6-8: CI/CD gates + observability for new modules
  • Clean-slate architecture decisions

Brownfield Track

For teams working with existing enterprise codebases.

  • Weeks 4-6: Heavier docs + analysis + safe PR selection

The Labs

Five hands-on labs that build your AI-assisted development skills progressively.

API Foundations

Lab 01

Build a minimal API in a safe sandbox environment. This lab works even when enterprise approvals are slow, teaching the core workflow: plan mode, task decomposition, permissions boundaries, diff-first development, and PR narration. You'll build a "Pet Registry" API - intentionally simple to focus on clean workflow rather than feature richness.

You'll Learn

  • Plan mode and task decomposition techniques
  • Permissions boundaries (what the tool can/can't do)
  • Diff-first development with human review loops
  • PR narration: what/why/risk/how-tested format

Deliverables

  • API contract (OpenAPI YAML or typed interface)
  • Plan-mode output with acceptance criteria
  • Implementation (minimum 2 endpoints)
  • Unit tests (4+ including edge cases)
  • PR narration + permissions note
Time: 4 hours
Week: 1
Languages: .NET / Java / Python

Brownfield Inspection & Documentation

Lab 02

Enterprise teams live in brownfield. This lab teaches how to inspect a larger repo safely (read-only when needed), generate usable documentation (not fluff), and create a risk-ranked backlog of safe improvements. You'll be dropped into a "typical enterprise service" repo and produce clarity: architecture overview, key boundaries, and change-risk heatmap.

You'll Learn

  • Safe repo inspection techniques (read-only first pass)
  • Dependency and boundary mapping
  • Hot spot identification: high churn + low tests + complexity
  • Creating actionable documentation for new engineers

Deliverables

  • Architecture README (what it is, how it runs, risks)
  • ADR v1 (one major decision with trade-offs)
  • Dependency & boundary map notes
  • Risk-ranked backlog (top 10 safe PR candidates)
  • "Safe PR" pick with plan + acceptance criteria
Time: 4 hours
Week: 4
Mode: Read-only analysis

Testing & Coverage Acceleration

Lab 03

AI is most valuable when it improves confidence. This lab teaches baseline measurement, high-ROI test selection, coverage delta reporting, and keeping AI-generated tests maintainable. Pick a module with weak coverage and improve it without introducing flakiness. Tests must be readable, deterministic, and avoid massive unmaintainable snapshots.

You'll Learn

  • Baseline coverage measurement techniques
  • High-ROI test target selection (high risk/high churn)
  • Human review loop for AI-generated tests
  • Coverage delta reporting and communication

Deliverables

  • Coverage baseline (before)
  • Test plan (what to test and why)
  • New tests (8+ including edge cases)
  • Coverage delta report (after)
  • Refactoring notes (if testing required changes)
Time: 4 hours
Week: 3
Focus: Unit tests preferred

Observability, Security & Compliance

Lab 04

Enterprise AI SDLC fails when teams ignore safety and operational reality. This lab has two parts: first, instrument your service with minimum viable observability (structured logs + correlation IDs). Then run a security/compliance drill covering prompt redaction and injection defense. This is where Fundamentals safety knowledge becomes practical.

You'll Learn

  • Structured logging with defined schemas
  • Correlation ID propagation end-to-end
  • Runbook creation for operational handoff
  • Prompt injection attack and defense in practice

Deliverables

  • Structured logging implementation
  • Correlation ID propagation
  • Runbook (local run + troubleshooting + log fields)
  • Redaction checklist + prompt review workflow
  • Prompt-injection drill findings + mitigations
Time: 4 hours
Weeks: 8-9
Parts: Observability + Security drill

UI & E2E Automation

Lab 05 (Optional)

For teams needing full-stack workflows and regression automation. Build a minimal UI for your API, then create browser-based E2E tests with Playwright or Cypress. This lab is intentionally flexible - pick your tech stack (React/Vue/Blazor) and testing framework to align with client standards.

You'll Learn

  • AI-assisted UI scaffolding
  • Browser-based E2E test creation
  • CI integration for E2E smoke tests
  • E2E failure triage patterns

Deliverables

  • UI (minimal, functional)
  • E2E test suite (minimum 3 tests)
  • CI job for E2E tests (if CI available)
  • E2E runbook and failure triage notes
Time: 4-6 hours
Week: Capstone extension
Stack: Flexible (React/Vue/Blazor + Playwright/Cypress)

Assessment & Rubrics

Every deliverable is evaluated against clear rubrics

PR Rubric

  • Clear "what changed" summary
  • Explained "why" with context
  • Risk assessment included
  • Testing approach documented
  • Follow-ups identified

Tests Rubric

  • Readable and well-named
  • Deterministic (no flakiness)
  • Edge cases covered
  • Maintainable structure
  • Coverage targets met

Docs Rubric

  • Actionable for new engineers
  • Accurate to current state
  • Risks clearly identified
  • Run/build steps verified
  • Trade-offs explained

Prompts Rubric

  • Clear constraints defined
  • Output contract specified
  • No sensitive data included
  • Reproducible results
  • Human review points identified
