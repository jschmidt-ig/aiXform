Lab 1 — API Foundations (Safe Sandbox)

Go from spec → plan → diff → PR narration in a small service.

  • Design an API contract and acceptance criteria
  • Use plan mode for decomposition + risk notes
  • Implement endpoints + tests + clean PR write-up

Lab 2 — Codebase Analysis + Doc Generation

Turn a production codebase into usable docs without changing logic.

  • Generate architecture README + ADR
  • Dependency/boundary mapping + risk heatmap
  • Backlog of “safe PR” candidates

Lab 3 — Testing + Coverage Acceleration

Increase confidence: add tests strategically and prove improvement.

  • Baseline coverage and identify weak modules
  • Generate tests + edge cases (human-reviewed)
  • Coverage delta report + quality rubric

Lab 4 — Observability + Security/Compliance Drills

Add instrumentation and harden prompts & workflows for enterprise risk.

  • Structured logs + correlation IDs
  • Redaction drills + prompt review workflow
  • Runbook + “what we changed / why / how tested”
What you can measure

PR throughput, test coverage delta, docs completeness, time-to-first-safe-PR.

Who it’s for

Software engineers and infra/data engineers shipping in regulated enterprise environments.

10-week program; 1h training + 2x lab blocks + live Q&A + office hours.
