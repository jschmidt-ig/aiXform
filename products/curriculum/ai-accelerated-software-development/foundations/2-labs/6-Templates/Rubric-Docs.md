# Rubric — Documentation (Architecture README / ADR / Runbook)

Score each dimension 0–4.

## Dimensions
1) **Clarity**
- 0: unclear, vague
- 2: understandable with gaps
- 4: clear and direct; low ambiguity

2) **Completeness**
- 0: missing major sections
- 2: covers basics
- 4: covers runtime, boundaries, risks, next steps

3) **Actionability**
- 0: theoretical
- 2: some steps
- 4: new engineer can run, debug, and contribute

4) **Accuracy**
- 0: wrong
- 2: mostly right
- 4: consistent with code and current state

5) **Security awareness**
- 0: ignores sensitive zones
- 2: mentions constraints
- 4: explicit sensitive zones + redaction guidance

## Required evidence
- Architecture README with: purpose, structure, run steps, risks, safe PR candidates
- ADR that captures decision + trade-offs + status
- Runbook with troubleshooting steps
