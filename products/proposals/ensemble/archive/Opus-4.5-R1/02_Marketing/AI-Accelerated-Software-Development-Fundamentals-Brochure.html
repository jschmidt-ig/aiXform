Program Overview

A lecture-only fundamentals program establishing the mental model for LLMs, context engineering, agentic workflows, tool calling, and security.

Format

20 independent sessions (lectures and demonstrations), each 20-30 minutes. Designed for flexibility - take them at your own pace.

Suggested Pace

2 sessions per day over 10 business days. Complete the full program in just 2 weeks.

No Code Required

Pure lecture format - no development environment needed. Labs (Weeks 3-6) and Applied (Weeks 7-10) come next.

Assessment

End-of-module test with critical safety pass/fail gates. Certification upon completion.

Who This Is For

Target Audience

  • Software engineers looking to understand AI-assisted development
  • Platform engineers building AI tooling and infrastructure
  • Technical professionals preparing for hands-on labs
  • Team leads evaluating AI coding tools for their teams
  • Anyone who will use CLI agents, API calls, or tool-based workflows

What You'll Learn

By completing this program, you will be able to:

  • ✓ Explain what an LLM does at inference time and what it cannot do (no in-session learning, no durable memory)
  • ✓ Understand training vs inference and why "crystallized intelligence" matters for how you use models
  • ✓ Master context windows and how context assembly affects quality and safety
  • ✓ Distinguish chatbot UI vs raw API usage vs agentic harnesses (IDE, CLI, cloud agent)
  • ✓ Explain agentic workflows (plan-act-observe-reflect) and where they break in practice
  • ✓ Understand tool calling, MCP, and when each approach is justified
  • ✓ Apply safe judgment on tool permissioning and least privilege principles
  • ✓ Recognize and mitigate prompt injection attacks

Foundation & Environment

Orientation and enterprise constraints

Course Orientation

  • The 3-layer stack: model, harness, workflow/governance

Tooling & Enterprise Constraints

Face the reality of enterprise AI adoption. Understand why approvals take time, what data can never enter a prompt, and how to work productively within security boundaries. Learn the "off-network coaching" model for safe knowledge transfer.

  • Approved models vs prohibited tools in enterprise

  • Security review processes and why they exist
  • The off-network coaching stance for sensitive environments

LLM Core Concepts

How language models actually work

Session 03

LLMs & Tokens

Demystify what happens when you send a prompt. Learn that LLMs predict the next token, not the next word, and why this distinction matters for cost, limits, and failure modes. Understand hallucination and format drift at a mechanical level.

You'll Learn
  • Next-token prediction as the core inference mechanism
  • Why tokens differ from words (subword effects)
  • Root causes of hallucination and overconfidence
  • Format drift and why output contracts exist
20-30 min Lecture + Demo
Session 04

Transformer Mental Model

Build intuition for how transformers process text without getting lost in the math. Understand embeddings, attention, and output distributions at a conceptual level. Learn why attention helps with dependencies but isn't reasoning.

You'll Learn
  • Token stream to output: embeddings, attention, layers
  • Why transformers handle long dependencies "somewhat"
  • Attention is not reasoning: key distinction
  • Where errors come from: missing vs conflicting context
20-30 min Lecture + Diagrams
Session 05

Training vs Inference

Understand the fundamental split between training (weight updates) and inference (fixed weights). Grasp "crystallized intelligence" and why capabilities are largely frozen. Learn what fine-tuning actually changes and what it can't.

You'll Learn
  • Training updates weights; inference uses fixed weights
  • Crystallized intelligence: capabilities frozen at training
  • Fine-tuning changes behavior style, not core limits
  • Misconception busting: "it learned from my prompt" (no)
20-30 min Lecture
Session 06

Context Window & Memory

Master the mechanics of context windows. Learn about truncation, recency bias, and "lost instructions." Understand why LLMs have no durable memory and why context engineering is a real, valuable skill.

You'll Learn
  • Context window limits and what happens at boundaries
  • Recency bias and why early instructions disappear
  • Memory is re-provided context, not internal state
  • Context packing patterns: headings, order, brevity
20-30 min Lecture + Demo

Prompting & Context Engineering

The craft of communicating with models

Session 07

Prompting Fundamentals

Learn the anatomy of effective prompts. Master the components: role, constraints, examples, acceptance criteria, and output contracts. Understand prompt hygiene and how to write prompts that produce testable, consistent outputs.

You'll Learn
  • Prompt anatomy: role, constraints, examples, criteria
  • Output contracts: sections vs JSON schema
  • Prompt hygiene: clarity, minimal ambiguity
  • Before/after: transforming vague prompts into structured ones
20-30 min Lecture + Examples
Session 08

Chatbot vs Raw Model Call

Peek behind the curtain of chatbot interfaces. Learn what gets hidden: system messages, safety wrappers, memory features. Understand the roles/messages mental model and why API-level thinking is essential for building agents.

You'll Learn
  • What chatbot UIs hide from you
  • System/developer/user message roles
  • Context assembly: what's really sent to the API
  • Why API-level thinking matters for agents
20-30 min Lecture + Diagrams
Session 09

Prompt Contracts & Output Schemas

Understand why contracts matter for automation, grading, reproducibility, and safety. Learn structured output patterns: sections, key-value pairs, JSON schemas. Practice validating outputs without running code.

You'll Learn
  • Why contracts enable automation and safety
  • Structured output patterns: sections, JSON, key-value
  • Contract examples: PR descriptions, risk checklists, change plans
  • Validation checklists for contract compliance
20-30 min Lecture + Examples
Session 10

Context Engineering Workshop

Master the art of constructing the "working set" - what to include, what to omit, how to order content. Learn compression patterns, when to invert conventional ordering, and how to avoid context poisoning from untrusted content.

You'll Learn
  • Working set construction: include/omit decisions
  • Ordering patterns: summary-first, constraints-first, examples-last
  • Context poisoning: isolating and labeling untrusted content
  • Context budget: what to spend tokens on
20-30 min Workshop

Agents & Harnesses

From chatbots to autonomous workflows

Session 11

Agents vs Workflows

Define what an agent actually is: LLM + tools + loop + objectives. Understand the agentic loop (plan-act-observe-reflect-iterate) and identify the specific points where agents commonly fail.

You'll Learn
  • Agent definition: the four required components
  • The agentic loop: plan, act, observe, reflect, iterate
  • Common failure points: mis-specified goals, tool misuse
  • Safe loop guardrail checklist
20-30 min Lecture + Diagrams
Session 12

Harness Taxonomy

Compare the major harness types: chatbot, IDE assistant, CLI agent, cloud agent. Understand what differs across tool surface, context assembly, permissions, and auditability. Learn why this course emphasizes CLI-first approaches.

You'll Learn
  • Four harness types and their characteristics
  • Tool surface comparison: what each can access
  • Permission and auditability differences
  • Why CLI-first matters for enterprise adoption
20-30 min Lecture + Comparison
Session 13

Agentic Coding Method

Learn the operational workflow that makes AI coding effective: plan, diff, test, narrate. Understand how agentic harnesses differ across environments and why workflow design matters more than any single tool.

You'll Learn
  • The plan-diff-test-narrate workflow
  • PR narration templates and why they matter
  • Harness differences: Claude Code CLI vs IDE vs browser
  • The "do not let the agent deploy" principle
20-30 min Lecture + Templates
Session 14

Model Mixing & Selection

Learn why mixing models across tasks is not only possible but often optimal. Understand when to use "bigger thinking" models vs "smaller fast" models. Navigate the trade-offs: cost, latency, reliability, safety.

You'll Learn
  • Why model mixing works (no durable memory)
  • Big-thinking vs small-fast model selection
  • Trade-off framework: cost, latency, reliability, safety
  • Simple decision tree for model selection
20-30 min Lecture + Decision Tree

Tool Calling & Integration

Extending model capabilities safely

Session 15

Tool Calling

Understand what tool calling actually is: structured function invocation via the harness. Learn why tool calling dramatically expands what models can do, and why that expansion introduces significant new risks.

You'll Learn
  • Tool calling as structured function invocation
  • How tool calls differ from "shelling out" or copy/paste
  • Capability expansion: what tools unlock
  • Risk expansion: why more capability means more danger
20-30 min Lecture + Examples
Session 16

MCP vs API Calls

Compare direct API calls with the Model Context Protocol (MCP). Understand when MCP's standardized interoperability and multi-step coordination are justified vs when they add unnecessary complexity.

You'll Learn
  • Direct API calls: point-to-point model invocation
  • MCP: standardized tool interoperability
  • When MCP is justified vs overkill
  • Decision framework for integration approach
20-30 min Lecture + Comparison

Security & Safety

Critical knowledge for safe deployment

Session 17

Securing Tool Calls

Master the principles of least privilege for AI tools. Learn about allowlists, scopes, and approval gates. Understand auditability requirements: logs, transcripts, and answering "who did what when."

You'll Learn
  • Least privilege: allowlists, scopes, approval gates
  • Permission sets: read-only, read+test, write, deploy
  • Auditability: logs, transcripts, accountability
  • Preventing untrusted inputs from driving tool calls
20-30 min Lecture + Examples
Session 18

Prompt Injection

Understand the most critical security threat in AI systems. Learn what injection is, recognize common patterns (in documents, web pages, repo text), and apply mitigations: input isolation, instruction hierarchy, gating, validation.

You'll Learn
  • Injection definition: untrusted text overriding instructions
  • Common injection patterns and where they hide
  • Mitigations: isolation, hierarchy, gating, validation
  • How to respond when you detect an injection attempt
20-30 min Lecture + Examples
Session 19

Security Scenarios Lab

Apply your security knowledge to realistic scenarios. Analyze 2-3 cases where you must pick safe permissions, identify injection attempts, and choose appropriate mitigations. Reinforce the critical safety principles tested in the final assessment.

You'll Learn
  • Scenario analysis: permission selection
  • Scenario analysis: injection identification
  • Scenario analysis: mitigation selection
  • Critical safety principles that are hard-fail on the test
20-30 min Interactive Lab
Session 20

Integrated Capstone

Walk through a complete end-to-end scenario that integrates everything: raw API call, harness behavior, tool calling, injection attempt, and mitigations. Final recap of key concepts and open Q&A.

You'll Learn
  • End-to-end integration: API to harness to tools
  • Security in practice: injection + permissioning
  • Full recap of Fundamentals program
  • Preparation for Labs & Applied phases
20-30 min Demo + Q&A
