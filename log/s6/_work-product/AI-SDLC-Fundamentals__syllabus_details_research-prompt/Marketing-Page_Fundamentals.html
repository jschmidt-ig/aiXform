AI SDLC Fundamentals

A two-week, lecture-only module that gives engineering teams a shared mental model for LLMs, context engineering, agentic workflows, tool calling/MCP, and the security risks (permissions + prompt injection) that matter before anyone runs tools.

Who it’s for

  • Software engineers and platform engineers preparing to adopt agentic coding workflows.
  • Technical professionals who need a clear, enterprise-safe foundation before labs/applied work.

Format

  • 20 micro-lectures (20–30 minutes each) over 2 weeks.
  • Command snippets and examples are shown on slides; learners do not execute tools during Fundamentals.
  • Ends with a short test with “critical safety” pass/fail.

Time commitment

~8–9 hours lecture
12 min end test
5–7 min evaluation

Typical schedule: 2 sessions/day, Mon–Fri for 2 weeks.

Expected outcomes

  • Explain tokens, inference, and why models don’t “learn” in-session.
  • Understand context window limits and how context assembly changes results.
  • Distinguish chatbot UIs from API calls and agentic harnesses (IDE/CLI/cloud).
  • Understand tool calling and when it changes capability.
  • Understand MCP at a conceptual level and when it’s justified vs direct APIs.
  • Apply safe judgment: least-privilege tool permissioning and prompt injection mitigation.

Assessment

End test (10–15 minutes)

  • 10 questions: comprehension + scenario judgment.
  • Pass: ≥80% and no misses on “critical safety” questions.

Evaluation questionnaire (5–7 minutes)

  • Confidence, clarity, and applicability.
  • Identifies blockers (tooling approvals, policy, time, lack of examples).

Notes

This Fundamentals module is intentionally lecture-only to fit enterprise constraints and keep the first two weeks unblocked by tool approvals. Labs and applied exercises follow later once tooling, permissions, and environment access are ready.
