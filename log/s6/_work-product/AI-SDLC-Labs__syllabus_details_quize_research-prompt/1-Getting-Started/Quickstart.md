# Quickstart: Week 1 Delivery (API Foundations Lab)

## Goal for Week 1
Get every participant from zero → **a small working service** with:
- an API spec
- a plan-mode implementation plan
- at least 2–4 unit tests
- a clean PR description (what/why/risk/how-tested)
- a basic security/permissions “tooling boundary” write-up

## 90-minute live session outline (Week 1)
1) **10 min** — Context + enterprise constraints + “what good looks like”
2) **15 min** — Demo: plan mode → task decomposition → acceptance criteria
3) **25 min** — Demo: implement 1 endpoint end-to-end + tests
4) **10 min** — Demo: PR narration template + risk notes
5) **20 min** — Participants start the lab
6) **10 min** — Q&A + submission expectations

## Participant workload (Week 1)
- 1h training video/lecture episode(s)
- 2h lab block #1
- 1h live demo/Q&A
- 2h lab block #2 / office hours
- 1h submit + self-review

## What you collect
- API spec (OpenAPI or equivalent)
- Implementation plan (plan-mode output)
- PR + PR narration
- Test report/coverage output (even if simple)
- Short “permissions + data handling” note

## Scoring
Use these rubrics from `6-Templates/`:
- PR rubric (delivery hygiene, risk, clarity)
- Tests rubric (edge cases, maintainability)
- Prompts rubric (constraints, output contract, redaction)
- Docs rubric (clarity and completeness)

## Week-1 fallback if tools aren’t approved
If the CLI/agent tool isn’t approved yet:
- Run the lab in a **local-only sandbox repo**
- Use “manual plan mode” prompts inside an approved chat UI
- Focus on: spec slicing, acceptance criteria, PR narration, tests
- Delay agent CLI mechanics to Week 2–3
